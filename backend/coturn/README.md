# coturn (TURN/STUN) Configuration

This directory contains the coturn configuration for Backroom's WebRTC infrastructure.

## What is TURN/STUN?

- **STUN** (Session Traversal Utilities for NAT): Helps peers discover their public IP addresses
- **TURN** (Traversal Using Relays around NAT): Relays media when direct P2P fails (strict firewalls, symmetric NAT)

Most WebRTC connections (~85%) work with STUN alone. TURN is a fallback for difficult network conditions.

## Ports

| Port | Protocol | Purpose |
|------|----------|---------|
| 3478 | TCP/UDP | STUN/TURN (standard) |
| 3479 | TCP/UDP | Alternative (some networks block 3478) |
| 5349 | TCP/UDP | TURNS (TURN over TLS) |
| 49152-65535 | UDP | Relay ports (media traffic) |

## Configuration

The main config is in `turnserver.conf`. Key settings:

### Authentication

Two modes are supported:

1. **Static credentials** (development):
   ```
   user=backroom:backroom_turn_secret
   ```

2. **Time-limited credentials** (production - recommended):
   ```
   use-auth-secret
   static-auth-secret=YOUR_SECRET
   ```
   The signaling server generates temporary credentials using HMAC.

### Generating Ephemeral Credentials

For production, use time-limited credentials generated by your signaling server:

```javascript
// Node.js example
const crypto = require('crypto');

function generateTurnCredentials(secret, ttlSeconds = 3600) {
  const timestamp = Math.floor(Date.now() / 1000) + ttlSeconds;
  const username = `${timestamp}:backroom`;
  const hmac = crypto.createHmac('sha1', secret);
  hmac.update(username);
  const credential = hmac.digest('base64');
  
  return {
    username,
    credential,
    ttl: ttlSeconds,
    urls: [
      'stun:your-server.com:3478',
      'turn:your-server.com:3478',
      'turns:your-server.com:5349'
    ]
  };
}
```

```go
// Go example
func generateTurnCredentials(secret string, ttlSeconds int64) (string, string) {
    timestamp := time.Now().Unix() + ttlSeconds
    username := fmt.Sprintf("%d:backroom", timestamp)
    
    h := hmac.New(sha1.New, []byte(secret))
    h.Write([]byte(username))
    credential := base64.StdEncoding.EncodeToString(h.Sum(nil))
    
    return username, credential
}
```

## Running Locally

```bash
# Start coturn
docker-compose up -d coturn

# Check logs
docker-compose logs -f coturn

# Verify it's running
docker exec backroom-coturn turnadmin -l
```

## Testing TURN

### Using Trickle ICE

1. Go to https://webrtc.github.io/samples/src/content/peerconnection/trickle-ice/
2. Add your TURN server:
   - URL: `turn:localhost:3478`
   - Username: `backroom`
   - Credential: `backroom_turn_secret`
3. Click "Gather candidates"
4. You should see `relay` candidates (indicates TURN is working)

### Command Line Test

```bash
# Test STUN
turnutils_uclient -t -u backroom -w backroom_turn_secret localhost

# Test with verbose output
turnutils_uclient -t -u backroom -w backroom_turn_secret -v localhost
```

## Production Considerations

### 1. External IP

For production, you MUST set the external IP:

```conf
# In turnserver.conf
external-ip=YOUR_PUBLIC_IP

# Or for cloud instances with private/public IP:
external-ip=PUBLIC_IP/PRIVATE_IP
```

### 2. TLS Certificates

For TURNS (secure), add certificates:

```conf
cert=/etc/ssl/certs/turn.pem
pkey=/etc/ssl/private/turn.key
```

### 3. Firewall Rules

Ensure these ports are open:
- 3478 TCP/UDP (STUN/TURN)
- 5349 TCP/UDP (TURNS)
- 49152-65535 UDP (relay range - can be narrower)

### 4. Bandwidth & Cost

TURN relay traffic can be expensive. Monitor:
- Number of relay allocations
- Bandwidth usage per relay
- Relay vs. P2P ratio

Aim for <15% TURN usage. If higher, investigate client network configs.

### 5. Scaling

For high traffic:
- Run multiple coturn instances behind a load balancer
- Use geographic distribution (deploy near users)
- Consider managed TURN services (Twilio, Xirsys) for global coverage

## Troubleshooting

### "401 Unauthorized"
- Check credentials match between client and server
- For time-limited creds, verify timestamp hasn't expired

### No relay candidates
- Verify coturn is running: `docker-compose ps coturn`
- Check firewall allows UDP on relay ports
- Ensure external-ip is set correctly

### Poor call quality over TURN
- Check relay port range is sufficient
- Monitor CPU/bandwidth on TURN server
- Consider upgrading or adding more instances

## Security Notes

- Never expose coturn admin CLI to the internet
- Rotate `static-auth-secret` regularly (at least every 90 days)
- Use TLS (TURNS) in production
- The config denies relaying to private IP ranges (prevents abuse)
